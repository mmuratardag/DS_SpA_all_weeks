Notes from the Software Engineering session:
Lines of code
< 10           no structure
< 100          functions
< 1000         functions + split into multiple .py files (modules)
< 10000        classes + multiple folders with modules (packages)
< 100000       probably multiple packages and/or repositories + lots of documentation
> 1 Mio        probably not a Python project
Terminology:
module  : .py file that you can import
package : folder with modules that you can import
library : something installable (preferably by pip)
